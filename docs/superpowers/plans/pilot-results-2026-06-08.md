## 1. 검증이 작동했다 (파일럿의 핵심 증명)

- 🔴 **고배당 9% = 오류 확정.** 원본 HTML "분리과세 9%, 2025 신설" → 실제 = **조특법 §104조의27, 2025.12.23 신설, 2026.1.1 시행, 누진 분리과세 14/20/25/30%**(과표 2천만/3억/50억 구간, 지방세 별도), 3년 한시. 감수의 "누진" 추정이 맞았고 구체 구간·시행일까지 확정.
- 다수 claim에 **누락 요건/예외 발견 → 교정**:
  - 식대: 현금 20만원은 "현물 미제공자 한정", 현물+현금 동시수령 시 현금 과세.
  - 자가운전: "시내출장 실제여비 대신 사규 지급" 요건 누락.
  - 납부지연가산세: 50%/10% 한도 누락.
  - 복권 200만: 슬롯머신/승마 등 호별 기준 구분 필요.
  - 성과급: "개인별 확정 연도"는 계량+비계량 평가 케이스에만 해당. "노동법 임금성 별개"는 1차 출처 미존재(강의기반).

## 2. 중대 구조 리스크 (P0에서 해소 필수)

**`law.go.kr`·홈택스 동적 로딩 → 조문 원문 직접 매칭 거의 전부 실패.** "확정" 8건 중 실제 1차 원문 직접 확인은 4건(`primarySourceVerified: true`)뿐, 나머지는 casenote·증권사·로펌 등 2차 교차확인 의존. → **`확정`이 과대평가될 수 있음.**

대응(P0 표준화): 법령 1차 원문 우회 확보 절차 = ① law.go.kr 인쇄용 URL(`...&print=print`) ② 조문 PDF 캡처 ③ casenote 등 2차는 보조. `primarySourceVerified` 플래그로 1차 직접확인 여부를 명시 분리.

## 3. 공수 실측 (estimateConfidence: low)

- 평균 **15.2분/fact**, 중앙 15분. 단 표본 편중(easy 7 / medium 3 / hard 0) → 전체 모집단 평균 **18~22분** 보수 상향.
- 전체 콘텐츠(fact 300~500 가정) 인간 검증 = **6~18주(중앙 10~13)**, 주 10~15시간 단일 검토자 기준.
- 권장 fact 상한 **350**(설계 500에서 하향).
- sourceType 분포: LAW 6 / NTS 3 / INTERPRETATION 1. verifyStatus: 확정 8 / 확인필요 2 / 강의기반 0.

## 4. 스키마 갭 9개 (실검증 도출 → schema.ts 반영)

| 필드 | 이유 |
|---|---|
| `primarySourceVerified` (bool) | 1차 원문 직접확인 vs 2차 교차확인 분리(law.go.kr 동적로딩 문제) |
| `subordinateLawRef` | 본법은 확인되나 시행령 조문 미확정 다수(시행령 §186조의2/§12-3/§17의2) |
| `confidenceScore` (0-100) | 확정 안에서도 신뢰도 차 큼 → 3단계 입도 부족 |
| `scopeLimitations` | P2P 14%, 현물+현금 과세, 출장여비 별도정산 과세 등 단서 분리 |
| `localTaxRef` | 지방소득세(15.4% 등) 근거가 소득세법 lawRef와 섞임 |
| `supersededRefs` | 동일 조번호 폐지 구제도 혼입(구 고배당 14~35%) 추적 |
| `appliesFrom` ≠ `effectiveDate` | 지급분 적용개시일 vs 법 전체 시행일(2026-04-21) 혼동 |
| `sunsetDate` | 3년 한시 등 종료조항 |
| `reviewerId`/`lastReviewedAt` | 재검토 주기·담당자 추적 |

## 5. P0 권장 계획

- **P0 = 감수 5건 + 신규 3장 ≈ 60~90 fact** → 검증 2~3주 + 통합 1~2주 = **첫 dogfood 약 3~5주**.
- 신규 3장은 `확인필요` 비율 높을 가능성 → 세무 전문가 검토 우선 배분 + law.go.kr 우회 절차를 P0에서 표준화해 P1 가속.
- P1 = 나머지 12장 fact 점진(4~13주).
