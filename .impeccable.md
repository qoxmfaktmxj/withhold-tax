# .impeccable.md

## Design Context

### Users
인사시스템(HR/급여) 개발사 **내부 실무자** — 고객지원(CS)이 고객 질문에 법적 근거를 찾을 때, 개발자가 원천징수 화면을 만들 때, 연말정산 프로젝트 팀이 참조할 때. **웹 전용**(데스크톱 브라우저), 모바일 미사용. 사용 맥락: 업무 중 빠르게 "이 규칙 맞나? 출처가 뭐지? 2026 기준 맞나?"를 확인하고 **국세청 재검증 없이 신뢰**하고 싶음.

### Brand Personality
**권위 있고(authoritative), 정직하고(honest), 읽기 좋은(legible).** 관보(官報)·법전의 신뢰감 + 잘 짜인 에디토리얼 매거진의 가독성. 차갑지 않고 따뜻한 종이 질감. "AI가 만든 대시보드"가 아니라 "잘 만든 실무 레퍼런스 출판물"로 느껴져야 함.

### Aesthetic Direction
- **방향 (2026-06-11 확정): 「AIRTABLE EDITORIAL」** — 사용자가 claude.ai Design으로 직접 설계한 시스템
  (export: `Downloads/원천징수 레퍼런스 디자인 시스템/readme.md`가 규칙 원문, `app/globals.css`에 동기화됨).
  이전의 크림+명조 세리프 "웜 에디토리얼"은 실물 비교 후 폐기 — 회귀 금지.
- **화이트 캔버스** + #dddddd 헤어라인. 틴트 배경 없음. **프라이머리 = 잉크 #181d26**(버튼·active nav·디스플레이),
  링크는 별도 블루 #1b61c9. 시그니처 컬러(coral #aa2d00·forest #0a2e0e·cream #f5e9d4)는 **풀블리드 카드/밴드 전용** — 작은 액센트 금지.
- **Pretendard Variable 단일 서체.** 디스플레이 400–500(볼드 금지 — 강조는 크기·명암대비), 소형 라벨만 600,
  숫자·조문·날짜는 tnum(tabular numerals). 별도 세리프·모노 서체 없음.
- 다크면 #181d26은 Formula·CheatCard·다크 밴드 전용. 푸터는 화이트 스트립 + mono 리비전.
- 레이아웃: **웹 전용 앱 셸** — 고정 좌측 사이드바(챕터 색인 + 검색 + 검증 범례) + 와이드 본문(content-max 1240px). 좁은 중앙 컬럼/모바일 느낌 금지.
- **신뢰층이 1급 디자인 요소**: 출처(법령 조문 + 시행일)는 법률 인용 라인, 검증상태는 작은 관인형 마크(확정/확인필요/강의기반).
- 챕터 카테고리 컬러 코딩(`--cat-*` + data-cat): 사이드바 번호·홈 카드 틱·챕터 eyebrow에만 — 절제 유지.

### Anti-references (이렇게 보이면 실패)
- 현재 빈약한 inline-스타일 프로토타입 (좁은 880px 중앙 컬럼, 작은 글씨, 밋밋한 색).
- 모바일 느낌. 카드 in 카드. border-left 색 줄무늬 콜아웃(금지). 영어 slug 화면 노출(meal-allowance 등 — 절대 금지, 한글 표시명 필수).
- 제네릭 SaaS 대시보드. cyan-on-dark, 보라 그라데이션, 그라데이션 텍스트.

### Design Principles
1. **출처가 곧 디자인** — 모든 검증 대상 사실에 법령 인용 + 시행일 + 검증상태. 장식이 아니라 신뢰의 본체.
2. **정직한 위계** — `확정`은 자신감 있게, `확인필요/강의기반`은 절제된 소수 표시. 미검증을 도배하지 않음.
3. **웹의 너비를 쓴다** — 고정 사이드바 + 와이드 본문. 좁은 컬럼·모바일 제약 없음.
4. **한글 우선** — 모든 화면 라벨은 한글 표시명. slug는 내부 식별자로만.
5. **잉크의 권위 + 여백의 절제** — 큰 Pretendard 라이트 디스플레이, 화이트 지면, 잉크 프라이머리, 시그니처 컬러는 풀블리드에서만.
